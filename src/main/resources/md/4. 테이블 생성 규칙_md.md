# 8. 데이터베이스(Database) 설계 및 생성 규칙

---

## 8.1 데이터베이스 종류

* PostgreSQL 사용

---

## 8.2 테이블 생성 규칙

### 테이블명 규칙

* 테이블명은 모두 대문자를 사용한다.
* 테이블 Prefix는 `TB_HKTMS_` 로 시작한다.

예시

```sql
TB_HKTMS_USER
TB_HKTMS_TEST_CASE
TB_HKTMS_DEFECT
TB_HKTMS_CODE
TB_HKTMS_CODE_GROUP
```

### 컬럼명 규칙

* 컬럼명은 모두 소문자를 사용한다.
* 약어 사용을 지양한다.

사용 금지 예시

```text
usr
plnr
mgr
dept_cd
```

권장 예시

```text
user_id
planner_id
manager_id
department_code
```

### PK 컬럼 규칙

* `seq`, `no` 사용 금지
* 반드시 `_id` 사용

사용 금지 예시

```text
user_seq
user_no
test_seq
```

권장 예시

```text
user_id
test_case_id
defect_id
```

### YN 컬럼 규칙

* PostgreSQL의 `BOOLEAN` 타입 사용을 우선 고려한다.

예시

```sql
is_active BOOLEAN
is_deleted BOOLEAN
is_test_available BOOLEAN
```

## 8.3 공통 감사(Audit) 컬럼 규칙

모든 테이블 생성 시 아래 컬럼을 반드시 포함한다.

| 컬럼명        | 설명     |
| ---------- | ------ |
| created_id | 생성자 ID |
| created_at | 생성일시   |
| updated_id | 수정자 ID |
| updated_at | 수정일시   |
| deleted_id | 삭제자 ID |
| deleted_at | 삭제일시   |

예시

```sql
created_id BIGINT
created_at TIMESTAMP
updated_id BIGINT
updated_at TIMESTAMP
deleted_id BIGINT
deleted_at TIMESTAMP
```

---

## 8.4 시퀀스 생성 규칙

### 시퀀스명 규칙

* 시퀀스명은 모두 대문자를 사용한다.
* Prefix는 `SEQ_` 로 시작한다.

예시

```sql
SEQ_HKTMS_USER
SEQ_HKTMS_TEST_CASE
SEQ_HKTMS_DEFECT
SEQ_HKTMS_HISTORY
```

---

## 8.5 인덱스 생성 규칙

### 인덱스명 규칙

* 인덱스명은 모두 대문자를 사용한다.
* Prefix는 `IDX_` 로 시작한다.

예시

```sql
IDX_HKTMS_USER_LOGIN_ID
IDX_HKTMS_TEST_CASE_STATUS
IDX_HKTMS_DEFECT_STATUS
```

---

## 8.6 제약조건 생성 규칙

### 제약조건명 규칙

* 제약조건명은 모두 대문자를 사용한다.
* Foreign Key Prefix는 `FK_` 로 시작한다.

예시

```sql
FK_HKTMS_TEST_CASE_USER
FK_HKTMS_DEFECT_TEST_CASE
FK_HKTMS_USER_ROLE
```

---

## 8.7 주석(Comment) 생성 규칙

### 주석 작성 규칙

* 모든 주석은 한글로 작성한다.
* 주석은 SQL 표준 주석(`--`)을 사용한다.

예시

```sql
-- 사용자 정보 테이블
-- 테스트 케이스 관리 테이블
-- 결함 관리 테이블
-- 공통 코드 그룹 관리 테이블
```

# 9. 공통 테이블 정의

## 9.1 공통 코드 그룹

테이블명

```sql
TB_HKTMS_CODE_GROUP
```

설명

* 공통코드 그룹 관리
* 테스트 상태
* 업무 분류
* 사용자 역할
* 결함 상태

등의 상위 코드 관리

---

## 9.2 공통 코드

테이블명

```sql
TB_HKTMS_CODE
```

설명

* 공통코드 상세값 관리
* 코드 그룹별 하위 코드 관리

# 10. 공통코드 관리 정책

## 10.1 테스트 상태 정의

코드 그룹

```text
TEST_STATUS
```

예시 코드

| 코드          | 설명  |
| ----------- | --- |
| READY       | 미진행 |
| IN_PROGRESS | 진행중 |
| SUCCESS     | 완료  |
| FAIL        | 실패  |
| HOLD        | 보류  |

---

## 10.2 결함 상태 정의

코드 그룹

```text
DEFECT_STATUS
```

예시 코드

| 코드           | 설명    |
| ------------ | ----- |
| ANALYSIS     | 분석중   |
| FIXING       | 조치중   |
| FIX_COMPLETE | 조치완료  |
| RETEST       | 재테스트중 |
| CLOSED       | 종료    |
