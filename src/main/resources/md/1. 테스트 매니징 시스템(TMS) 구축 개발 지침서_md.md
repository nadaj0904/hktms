# 테스트 매니징 시스템(TMS) 구축 개발 지침서

---

# 1. 프로젝트 개요

## 1.1 프로젝트명

테스트 매니징 시스템(Test Management System, TMS)

---

## 1.2 구축 목적

통합테스트(UAT/통합검증) 단계에서 발생하는 테스트 진행 현황, 결함(Bug) 현황, 일정 관리 및 고객사와 수행사 간 커뮤니케이션을 통합 관리할 수 있는 웹 기반 테스트 관리 시스템을 구축한다.

본 시스템은 프로젝트 참여자가 동일한 정보를 실시간으로 공유하여 프로젝트 품질을 향상시키고 테스트 완료율 및 결함 처리 현황을 체계적으로 관리하는 것을 목적으로 한다.

---

## 1.3 구축 목표

### 테스트 관리

* 테스트 케이스 등록 및 관리
* 테스트 수행 결과 관리
* 업무별 테스트 진행률 자동 산출

### 결함 관리

* 결함 등록 및 추적
* 결함 처리 상태 관리
* 재테스트 관리
* 결함 처리 이력 관리

### 프로젝트 현황 관리

* Dashboard 기반 실시간 현황 제공




각 역할별 업무 수행 지원

# 3. 시스템 기능 요구사항

## 3.1 Dashboard

### 목적

프로젝트 전체 테스트 현황과 결함 처리 현황을 실시간으로 확인할 수 있도록 한다.

---

### 화면 구성

#### 테스트 진행 현황

표시 항목

* 전체 테스트 건수
* 완료 건수
* 진행중 건수
* 실패 건수
* 보류 건수
* 미진행 건수

산출 공식

전체 테스트 진행률(%)

= 완료 건수 ÷ 전체 테스트 건수 × 100

예시

90건 ÷ 100건 × 100 = 90%

---

#### 업무별 테스트 진행 현황

표시 항목

* 업무명
* 전체 건수
* 완료 건수
* 실패 건수
* 진행률(%)

산출 공식

업무별 진행률(%)

= 완료 건수 ÷ 전체 건수 × 100

---

#### 결함 현황

표시 항목

* 전체 결함 건수
* 분석중
* 조치중
* 조치완료
* 재테스트중
* 종료

산출 공식

결함 처리율(%)

= 조치완료 건수 ÷ 전체 결함 건수 × 100
